Authentication

Bearer Token Authentication

All API endpoints require authentication via an API key passed in the HTTP header. Generate API keys from the Integrations Hub.

Authorization: Bearer YOUR_STRIPE_API_KEY_HERE
Note: API keys are shown only once at creation time. Store them securely on your side.
HeaderValue
AuthorizationBearer sk_live_...
Content-Typeapplication/json (for POST/PUT)
Products
GET /api/v1/products
List all products for your account.
# Response [ { "id": 1, "sku": "SKU-001", "name": "Product Name", "description": "...", "cost_price": 100.00, "selling_price": 150.00, "reorder_point": 10, "stock": 50 } ]
POST /api/v1/products
Create a new product.
# Request body { "sku": "SKU-002", "name": "New Product", "description": "Product description", "cost_price": 50.00, "selling_price": 75.00, "reorder_point": 5 } # Response (201 Created) { "success": true, "product_id": 2 }
GET /api/v1/products/:id
Get a single product with inventory details.
Inventory
GET /api/v1/inventory/:product_id
Get stock levels across all warehouses for a product.
[ { "warehouse_name": "Main Warehouse", "available_qty": 25 } ]
POST /api/v1/inventory/:product_id/adjust
Adjust inventory stock for a product.
# Request body { "warehouse_id": 1, "qty_change": 10, "move_type": "restock", "notes": "API restock adjustment" } # Response { "success": true, "product_id": 1, "adjustment": 10 }
move_type values: restock, sold, adjustment, transfer_in, transfer_out, api_adjust
